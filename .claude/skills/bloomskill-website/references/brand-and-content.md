# Brand & Content — bloomskilltech.in

This file contains the positioning logic and the approved copy for every core page. Use this copy as the base; light editing for fit is fine, inventing new claims is not.

## Positioning

**Category:** Digital product studio (not "IT services company", not "agency")
**Customer:** Indian startups, SMBs, and education/training businesses that need a product built — founders who would otherwise stitch together a freelancer designer + a dev shop.
**Enemy:** Fragmented delivery. The designer-to-developer handoff where projects lose months and money.
**Promise:** One team from strategy to shipped product.
**Reason to believe #1 (people):** The team includes graduates of IIT Bombay, IIM Indore and NID Ahmedabad — India's top engineering, business, and design schools, on one team.
**Reason to believe #2 (proof):** We build and run our own products (Trainerr.in, exam platforms) with the same stack and standards we sell.
**Reason to believe #3 (process):** Visible engineering discipline — automated testing, CI/CD, phased delivery, fixed-scope SOWs.

**Institute mention rule:** Name the institutes only as a collective team credential ("our team includes graduates of..."). Do NOT assign an institute to a role, a 4D stage, or an individual anywhere on the site.

## Voice

Confident, specific, slightly direct. Short sentences. Numbers over adjectives. Indian business context is fine (₹, GST, UPI, Razorpay) — don't write for a Silicon Valley audience. Never use: unlocking potential, transformative, innovative solutions, catalyzing, synergy, cutting-edge, world-class, passionate.

### The problem (short section)
**H2:** Most digital projects die in the handoff.
You hire a designer. Then a development agency. The design gets "adjusted" to fit what the developers can build. Deadlines slip while two vendors blame each other. You pay twice and launch late.

**H2 (the turn):** We removed the handoff.
At BloomSkill, the people who plan your product, design it, and build it sit on the same team, in the same standups, accountable to one timeline and one contract.

### The 4D Framework section
**H2:** How we work: the 4D framework

**1 — Strategy.**
Before a single screen is designed, we define what wins. Market and competitor mapping, user definition, scope, pricing and revenue logic, and a phased roadmap. *Deliverable: a product strategy document and fixed-scope SOW you own, whoever you build with.*

**2 — Design.**
Research-led UX and interface design. User flows, wireframes, high-fidelity UI, and a reusable design system — not just pretty screens, but screens engineered to convert and built to be built. *Deliverable: clickable prototype + complete design system in Figma.*

**3 — Develop.**
Disciplined engineering with modern, proven stacks (FastAPI, PostgreSQL, React, Rust where performance demands it). Every build ships with automated tests, code review, security hardening, and documentation. *Deliverable: production code with 80%+ automated test coverage and CI checks on every commit.*

**4 — Deliver.**
Launch is a process, not a hope. CI/CD pipelines, staged deployments, monitoring, performance budgets, training and handover docs — plus post-launch support so you're never stranded. *Deliverable: a live product, a runbook, and a team that picks up the phone.*

### Proof section
**H2:** We don't just build for clients. We build for ourselves.
The same team, stack and standards run our own products — so we feel every shortcut we'd be tempted to take.
- **Trainerr.in** — a two-sided marketplace for India's training ecosystem: escrow payments, GST invoicing, Razorpay integration.
- **Secure exam platform** — desktop anti-cheat examination system with camera proctoring and real-time risk scoring.
- **Commerce builds** — end-to-end e-commerce platforms with admin panels, courier API integrations and payment flows for Indian retail.

(Each links to its case study.)
